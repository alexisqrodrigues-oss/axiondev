# Bem-vindo ao axison.dev

Este é o primeiro post do blog. A ideia é simples: **documentar tudo** o que eu aprendo no universo dev e IA.

## Por que esse blog existe

Acredito que escrever é a melhor forma de **fixar** conhecimento. Muitas vezes resolvo um problema, esqueço a solução, e três meses depois preciso passar pela mesma dor de novo. Esse blog é meu antídoto contra isso.

> "_Se você não escreveu, você não sabe._"

## O que você vai encontrar aqui

- Workflows e dicas de produtividade
- Instalação e configuração de softwares
- Estudos acadêmicos resumidos
- Scripts úteis (bash, python, node)
- Automações com IA
- Tutoriais passo a passo
- Comparativos e benchmarks de modelos

## Como funciona tecnicamente

Tudo é **markdown puro**. Cada post vive em uma pasta com seu `README.md` e uma pasta `assets/` opcional. Sem CMS, sem banco de dados, sem complicação.

```bash
~/axison.dev/posts/
├── 2026-04-22_bem-vindo/
│   ├── README.md
│   └── assets/
└── 2026-04-21_setup-neovim/
    ├── README.md
    └── assets/
```

Quer escrever um post? Cria a pasta, joga o markdown dentro, commita. **Pronto.**
